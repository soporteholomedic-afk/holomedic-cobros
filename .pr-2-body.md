# Worker Detail Table — Archivos Slice (PR 2 of 2)

## Summary

This is the second PR in the `worker-table-aptitud-archivos` change. It
adds a per-ficha `Archivos` column to the `WorkerDetailTable` and a
modal that lists/downloads files from the LAN UNC share at
`\\172.16.10.12\sigla\{ruc}\{dni}\{idAten}`.

- **Target branch**: `feature/worker-table-aptitud-archivos-pr1` (PR 1,
  already landed the `Aptitud` column from the same change).
- **Base branch for the chain**: `feature/worker-table-aptitud-archivos`
  (tracker). PR 1 → PR 2 → final integration PR.
- **Chain strategy**: `feature-branch-chain`. The reviewer-facing diff
  for this PR is the slice below — it does not include the PR 1
  changes.

## What this PR does

1. **`IFileRepository` hexagonal port** in `domain/ports.ts` — list +
   stream — so the API routes never touch `fs` directly.
2. **`UncFileRepository` adapter** + **`getFileRepository` factory** +
   test seam (`__setFileRepositoryForTests`) under
   `infrastructure/files/`. Pure Node `fs` against the UNC path with
   `path.win32.join`; `ENOENT` → `[]`; per-file stat failures logged
   and skipped; `zipAll` returns a streaming `archiver` instance.
3. **Three API routes** under `src/app/api/files/`:
   - `GET /api/files/list?ruc&dni&idAten` — 200 with sorted files,
     400 on missing args or non-digit `dni`, 502 on UNC I/O error.
   - `GET /api/files/download?ruc&dni&idAten&filename` — streams the
     file with `Content-Disposition: attachment`. Two-layer path-traversal
     defense (`sanitizeDownloadName` + `path.resolve` containment),
     `ENOENT` → 404, 50 MB warn log.
   - `GET /api/files/download-all?ruc&dni&idAten&nombrePaciente&empresa` —
     streams a server-side zip via `archiver` in `STORE` mode. Sanitized
     filename `{nombre} - {dni} - {empresa}.zip`. 502 on share unreachable.
4. **`usePatientFiles` hook** — empty-args short-circuit (no fetch),
   `refetch` reissues, 5 cases from REQ-FL-2 covered.
5. **`FilesModal` component** — state machine
   (`loading | empty | error | ready`), header + scrollable body + footer,
   extension-aware icon map (lucide-react), per-file download `<a>`,
   bulk-download `<a>` disabled in empty state and labeled
   `Generando zip...` while in flight. Escape / backdrop / X close.
6. **`WorkerDetailTable`** — adds the 10th `Archivos` column with a
   `Ver Archivos` button on every primary and sub-row. Modal state is
   keyed by `{ dni, fichaIndex }` so sub-rows do not share state.
7. **`src/lib/sanitize-filename.ts`** — `sanitizeComponent`,
   `sanitizeZipName`, `sanitizeDownloadName` (per design truth table).
8. **`archiver@^7.0.1`** + **`@types/archiver@^7.0.0`** added.
9. **`.env.local.example`** updated with `FILE_SERVER_BASE_PATH`
   (gitignore exception added so the example is tracked).
10. **`.gitignore`** — added `!.env*.example` so the env example
    files are not silently ignored.

## Spec ACs covered

  `(ruc, dni, idAten)`.
- **AC-7**: sub-rows for the same `dni` open independent modals via
  the `{ dni, fichaIndex }` state key.
- **AC-8**: worker-only row (no `idAten`, no `nroRuc`) renders the
  empty state in the modal.
- **AC-9**: `/api/files/list` returns 200 + sorted `FileEntry[]` or
  200 + `{ files: [] }`.
- **AC-10**: empty `ruc`/`dni`/`idAten` returns 400; non-digit `dni`
  returns 400.
- **AC-11**: `/api/files/download` streams the file with
  `Content-Disposition: attachment`.
- **AC-12**: `/api/files/download?filename=..\..\..\windows\sam`
  returns 400.
- **AC-13**: `/api/files/download-all` returns `Content-Type:
  application/zip` and a sanitized `Content-Disposition` filename.
- **AC-14**: zip filename `Juan/Pérez` produces `Juan_Pérez - ...`.
- **AC-15**: `normalizeCondic` truth table (already landed in PR 1;
  not regressed).
- **AC-16**: no test in the suite opens a connection to
  `\\172.16.10.12\sigla` — `UncFileRepository` tests mock `node:fs`
  at the module boundary, API route tests inject a mock
  `IFileRepository` via `__setFileRepositoryForTests`, component
  tests stub `usePatientFiles`.
- **AC-17**: all changed-file tests pass with `pnpm vitest run` on
  the files involved.

## Manual smoke checklist (operator must run before merge)

> Run on a Windows box with the share reachable.

1. Add `FILE_SERVER_BASE_PATH=\\172.16.10.12\sigla` to `.env.local`.
2. `pnpm dev`.
3. Open the consolidated-results page (`/envio-resultados` or
   wherever `WorkerDetailTable` is rendered).
4. Click `Ver Archivos` on a known ficha. The modal must list the
   files in the folder (sorted alphabetically).
5. Click one of the per-file `Descargar` links. The file must
   download with its original name and a content-type inferred from
   the extension.
6. Click `Descargar todos`. The zip must download as
   `{nombre} - {dni} - {empresa}.zip` (sanitized — illegal chars
   replaced with `_`, whitespace collapsed, empty components dropped).
7. For a worker-only row, click `Ver Archivos`. The modal must
   show the empty state `No hay archivos para esta ficha`.
